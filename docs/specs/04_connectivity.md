# 04. Коннектом (Connectivity)

> Часть архитектуры [Genesis](../../design_specs.md). Как нейроны находят друг друга, как связи рождаются, живут и умирают.

---

## 1. Правила Роста и Соединений (Connectivity)

### 1.1. Архитектура: Pub/Sub

| Роль | Аналогия | Поведение |
|---|---|---|
| **Аксон (Publisher)** | Радиостанция | Вещает сигнал в пространство (Active Tail + Global Buffer). Ему всё равно, слушает 0 или 1 000 000 дендритов. **Не хранит список слушателей** (это бы убило память). |
| **Сома/Дендрит (Subscriber)** | Радиоприёмник | Решает, на какой Axon ID «настроиться». Лимит каналов = 128 слотов. |

Ограничение в 128 — это лимит **входной пропускной способности** конкретного нейрона, а не выходной популярности аксона.

### 1.2. Классификация Аксонов (Axon Classes)

В `anatomy.toml` и `blueprints.toml` аксоны явно делятся на три класса, чтобы не считать лишнюю физику:

| Класс | Описание | Физика |
|---|---|---|
| **Local** (Интернейроны) | Локальные связи, облака торможения | Честная физика, коллизии |
| **Horizontal** (Слой) | Связи в плоскости XY | Честная физика в плоскости |
| **Projectors** (Белое вещество) | Дальние/межзональные связи | Только `Port Out` + `Port In` + `Delay_Counter` |

- **Оптимизация Projectors:** Симулируем только точку входа (в кабель) и точку выхода. Ствол аксона **не существует** в физике коллизий (экономим память и такты) — он существует только как `Delay_Counter`.

### 1.3. Аксоны (Axon Growth)

- **Цель (Target):** Не "вектор к центру" (избегать пучка в одной точке), а движение строго по Z (вверх/вниз) к целевому слою.
- **Глубина (Penetration):** Аксон должен не просто коснуться границы слоя, а проникнуть внутрь (на случайную или заданную глубину внутри слоя `axon_target_level`).
- **Остановка (Stop Condition):**
  - Рассчитывается относительное положение сомы в домашнем слое: `Soma_Rel_Z = (Soma_Z - Layer_Min) / Layer_Height`.
  - Аксон останавливается в целевом слое на той же относительной позиции: `Target_Z = Target_Layer_Min + (Soma_Rel_Z * Target_Layer_Height)`.
- **Ориентация:** Растет направленно, ориентируясь по другим сомам.
- **Дифференциация роста (H/V):**
  - Вертикальные нейроны (`growth_vertical_bias ≥ 0.5`): цель = целевой слой по Z. Остановка при достижении Target_Z.
  - Горизонтальные нейроны (`growth_vertical_bias < 0.5`): цель = радиальное направление в XY-плоскости домашнего слоя. Остановка при выходе из домашнего слоя по Z или исчерпании шагов.
- **Идентификация (Identification):** Каждый сегмент аксона хранит **полную 4-битную маску типа** нейрона-владельца (Type Index 0..15, см. [03_neuron_model.md §2](./03_neuron_model.md)).
  - Это позволяет дендритам применять специфические правила пластичности (разные Learning Rates) в зависимости от типа источника, без обращения к соме владельца.

### 1.4. Дендриты (Dendrite Connectivity)

- **Пространственная фильтрация:** Дендриты ищут аксоны только в определённой области.
- **Типовая фильтрация (Hard Filter):**
  - Дендрит проверяет полную маску типа владельца аксона (через Segment ID) до создания связи.
  - Правило: Не создавать синапсы с несовместимыми типами (whitelist/blacklist). Это предотвращает создание бесполезных связей.
- **Лимиты (Limits):**
  - **Max Synapses:** Максимум 128 дендритов на одну сому.
  - **Rule of Uniqueness:** Сома может иметь **только одну связь** с конкретным Axon ID. Дублировать сигнал бессмысленно — дефицитные слоты (128) лучше потратить на разные источники.

### 1.5. Компенсация Силы (Super-Synapse)

Отсутствие мульти-коннекта компенсируется широким динамическим диапазоном веса (`i16`: от -32768 до +32767). Знак бейкается при создании из `is_inhibitory` в `blueprints.toml`: (+) = Excitatory, (-) = Inhibitory. Абсолютное значение начального веса задаётся per-type через `initial_synapse_weight`.

- **Логика:** В момент, когда дендрит видит активный аксон: `Soma_Voltage += Synapse_Weight` (GPU: `v += (int32_t)dendrite_weights[col_idx]`, cast i16→i32 для безопасного суммирования 128 слотов).
- **Масштаб:**
  - Слабая связь (только родилась): Вес = `initial_synapse_weight` (по умолчанию `74`).
  - Сильная связь (обучилась): Вес = до `32 767` (i16 max). Добавляем `+32 767` к вольтажу.
- **Эквивалентность:** Одна прокачанная связь (32767) ≈ 443 слабых (74 × 443 ≈ 32767). Один «супер-синапс» математически заменяет пучок слабых.
- **Закон Дейла:** Знак синапса **неизменен** после создания. GSOP модифицирует только абсолютное значение: `new_abs = abs(w) + delta`, результат = `sign(w) × new_abs`. Clamp: `0 ≤ abs ≤ 32767`.
- **Почему это правильно:** В биологии 5 синапсов нужны для надёжности (вероятностный выброс медиатора). У нас физика **детерминированная** — 5 связей по 10 = 1 связь на 50.

### 1.6. Стратегия Выбора (Baking Heuristics)

При плотности 4–5% и длинных аксонах количество кандидатов на подключение превышает лимит (128). На этапе Baking (CPU) применяется эвристика выбора лучших кандидатов, а не рандом:

1. **Distance Bias:** Ближайшие аксоны приоритетнее (короткие связи энергетически дешевле).
2. **Type Matching:** Реализация через `dendrite_whitelist` (жёсткий фильтр) и `sprouting_weight_type` (мягкий скоринг).

#### 1.6.1. Sprouting Score (Горячие слоты, Night Phase)

Во время Night Phase (Sprouting) дендрит выбирает аксон-кандидат по тройному скорингу. Веса конфигурируются по зоне:

```rust
// genesis-baker/src/bake/sprouting.rs

/// Готовится 1 раз за Night Phase до Sprouting.
/// Использует веса из dendrite_weights[], уже скачанных с GPU (Step 2: Download).
fn compute_power_index(soma_id: usize, weights: &[i16], padded_n: usize) -> f32 {
    let mut power = 0u32;
    for slot in 0..128 {
        let w = weights[slot * padded_n + soma_id];
        power += w.unsigned_abs() as u32; // Без float, без бранчей
    }
    power as f32 / (128.0 * 32767.0) // нормализация 0.0..1.0
}

fn sprouting_score(
    dist: f32,
    target_power: f32, // compute_power_index() целевой сомы
    epoch_seed: u64,   // wyhash(dendrite_id, master_seed + night_epoch)
    cfg: &SproutingWeights,
) -> f32 {
    let dist_score  = 1.0 / (dist + 1.0);                   // Ближе → выше
    let power_score = target_power;                          // Сильные сомы магнитят
    let explore     = random_f32(epoch_seed);                // Ротация предпочтений по эпохе

    dist_score  * cfg.weight_distance
        + power_score * cfg.weight_power
        + explore     * cfg.weight_explore
}
```

| Компонент | Описание | Типичное значение |
|---|---|---|
| `weight_distance` | Ближайший аксон = лучше | 0.5 |
| `weight_power` | Мощные сомы (Главный хуб) притягивают больше | 0.4 |
| `weight_explore` | Шум / ротация по эпохе, чтобы не зацикливаться на одних | 0.1 |
| `weight_type` | Совпадение типа источника и цели (мягкий аттрактор) | 0.2 |

> **Самостабилизация:** `soma_power_index` = ×|weight| постоянно колеблется. Сома, потерявшая связи, становится менее привлекательной. Естественная насыщаемость защищена правилом: 1 дендрит = 1 сома + `max_dendrites = 128`.

### 1.7. Бесшовная Граница (Ghost Axons)

#### Механика

Аксон растёт пока не достигнет границы своего шарда (`ShardBounds`). При пересечении:

1. Генерируется **GhostPacket** — описание аксона в точке пересечения
2. Шард B получает пакет и вызывает `inject_ghost_axons()`
3. **Ghost Axon** продолжает рост оставшиеся `remaining_steps` с сохранением инерции (`entry_dir`)
4. Ghost Axon добавляется к общему списку аксонов шарда B и участвует в дендритном коннекте
5. Если Ghost Axon снова пересекает границу → генерируется новый пакет для шарда C

```rust
// genesis-baker/src/bake/axon_growth.rs

pub struct GhostPacket {
    pub origin_shard_id: u32,  // Откуда пришёл
    pub soma_idx: usize,       // Оригинальная сома (usize::MAX если внешний)
    pub type_idx: usize,       // Тип нейрона (для whitelist, affinity и т.д.)
    pub entry_x: u32,          // Точка входа в новый шард
    pub entry_y: u32,
    pub entry_z: u32,
    pub entry_dir: Vec3,       // Направление в момент пересечения (инерция)
    pub remaining_steps: u32,  // Сколько шагов осталось расти
}
```

#### Поведение Ghost Axon в шарде B

- **Нет `v_global`** — Ghost Axon не имеет новой цели, летит по инерции
- **Есть `v_attract`** — притягивается к нейронам шарда B (обычный Cone Tracing)
- **`soma_idx = usize::MAX`** — нет локальной сомы → GSOP не применяется (пластичность Ghost Axon идёт в шарде-источнике)
- **Дендриты цепляются** — сомы шарда B могут подключаться к Ghost Axon как к обычному

#### Монорежим (один шард)

`ShardBounds::full_world(&sim)` задаёт границы = весь мировой объём.  
Ghost Packets **никогда не генерируются**, поведение идентично текущему.  
При появлении пакетов — логируется предупреждение.

- **Телепорты:** Только для Global Connectivity (межзональные связи через Атлас, см. [06_distributed.md §3](./06_distributed.md)). Локальный рост — всегда бесшовный.

## 2. Пластичность и Математика (Plasticity & Math)

### 2.1. Вес Синапса

- **Тип:** `i16` (-32768..+32767). Знак определяет E/I (бейкается при создании связи).
- **Семантика:** Вес = сила тока со знаком. При срабатывании: `Soma_Voltage += Weight`. Знак (+/-) определяет E/I, бейкается при создании связи.
- **Инициализация:** Начальный вес задаётся в конфиге (LUT по Variant ID).

### 2.2. Триггер Обучения: Active Tail Check

- **Правило:** При спайке сомы проверяем, касается ли дендрит **активного хвоста** (Active Tail) сигнала в аксоне (см. [05_signal_physics.md](./05_signal_physics.md)).
- **Причинность:** Сигнал физически пришёл → сома выстрелила → GSOP сработал. Не «совпадение в тике», а проверка пространственного перекрытия.
- **Результаты:**

| Ситуация | Active Tail | Действие |
|---|---|---|
| Сома спайкует, дендрит касается хвоста | ✅ Да | `Weight += potentiation` (усиление) |
| Сома спайкует, дендрит НЕ касается хвоста | ❌ Нет | `Weight -= depression` (ослабление) |
| Вес падает ниже `1` | — | Связь удаляется (Pruning) |

### 2.3. GSOP через LUT (Variant-Dependent Learning)

Конкретные значения GSOP **не хардкодятся**. Они задаются в [02_configuration.md §6](./02_configuration.md) и загружаются в `__constant__` память GPU как LUT по Variant ID (биты 2–3 типа нейрона):

| Variant | Potentiation | Depression | Множитель | Профиль |
|---|---|---|---|---|
| `00` Regular | из конфига | из конфига | `1.0` | Стандартная пластичность |
| `01` Fast | из конфига | из конфига | `1.0` | Агрессивное обучение |
| `10` Stable | из конфига | из конфига | `0.1` | Медленное, устойчивое |
| `11` Fixed | из конфига | из конфига | `0.0` | GSOP заморожен (реле) |

```
// GPU (псевдокод)
let var_id = (type_bits >> 2) & 0x03;
let gsop = GSOP_LUT[var_id];
weight += gsop.potentiation * gsop.multiplier;
```

### 2.4. Кривые Инерции (Inertia Curves)

Вместо бинарной заморозки (Permanence bit) — **нелинейное сопротивление изменениям**.

- **Механика:** Текущий `abs(вес)` (`i16`) делится на **16 рангов** (по 2048 единиц). Для каждого ранга в конфиге задан свой коэффициент сопротивления.
- **Суть:** Чем сильнее связь, тем сложнее её изменить (как усилить, так и ослабить). «Детские» связи пластичны, «старые» — монументальны.

| Ранг | Диапазон abs(вес) | Коэффициент | Поведение |
|---|---|---|---|
| 0–3 | 0 – 8 191 | Высокий | Быстрое обучение, быстрая смерть |
| 4–7 | 8 192 – 16 383 | Средний | Стабилизация |
| 8–11 | 16 384 – 24 575 | Низкий | Трудно изменить |
| 12–15 | 24 576 – 32 767 | Минимальный | Почти монументальная связь |

```
// GPU (псевдокод)
let rank = abs(weight) >> 12;                // 16 рангов
let inertia = INERTIA_LUT[rank];           // Коэффициент из constant memory
let delta = gsop.potentiation * inertia;   // Чем выше ранг, тем меньше delta
weight = sign(weight) * clamp(abs(weight) + delta, 0, 32767);
```

**Почему не Permanence bit:** Бинарная заморозка создавала стеклянный потолок — связь, став «постоянной», теряла способность адаптироваться. Inertia Curves обеспечивают плавный переход от пластичности к стабильности без абсолютных границ.

### 2.5. Градиентная Стабильность (Slot-Based Decay)

128 дендритных слотов делятся на два класса с разным decay:

| Диапазон | Роль | Decay | Поведение |
|---|---|---|---|
| **Slots 0–79** | Long-Term Memory (LTM) | Низкий | Сюда попадают только проверенные связи |
| **Slots 80–127** | Working Memory (WM) | Агрессивный | Быстрый отсев гипотез |

- **Механизм:** В шейдере GSOP множитель `decay` зависит от индекса слота (индекс в массиве Columnar Layout).

### 2.6. Ночная Сортировка (Nightly Sort)

Во время фазы «Ночь» (CPU, см. [07_gpu_runtime.md](./07_gpu_runtime.md)) выполняется консолидация памяти:

1. **GPU Sort & Prune:** Segmented Radix Sort по `abs(weight)` descending. Слоты с `abs(w) < threshold` обнуляются.
2. **Promote:** Сильные связи из WM (80–127) физически переезжают в LTM (0–79).
3. **Evict:** Слабые связи вытесняются в хвост массива (WM) или удаляются, если вес < порога.
4. **Результат:** После ночи LTM содержит топ-80 самых сильных связей, WM — 48 свежих кандидатов.

## 3. Межшардовая Связность (Inter-Shard Connectivity)

Для внутренней логики нейрона не существует понятия «другой сервер». Граница шарда прозрачна благодаря абстракции Channel и механике Ghost Axons.

### 3.1. Унифицированный Интерфейс (The Channel Trait)

Шард не знает топологии кластера. Он взаимодействует только с интерфейсами (Channels) по 4 границам своей плитки (см. [06_distributed.md §1.2](./06_distributed.md)).

| Channel | Реализация | Механика |
|---|---|---|
| **Local** (Intra-node) | Shared Memory / Zero-Copy | Шард А пишет в буфер, Шард Б читает тот же адрес. L2-кеш работает на нас. Копирования данных нет. |
| **Network** (Inter-node) | TCP/UDP Stream | Сериализация → сеть → десериализация. Латентность компенсируется размером батча (`sync_batch_ticks`). |

### 3.2. Два Потока Данных (Fast & Slow Paths)

Связь разделена на два независимых потока для оптимизации пропускной способности.

**1. Геометрия (Slow Path / Growth)**

- **Частота:** Редко (только при обновлении топологии / росте аксонов).
- **Структура пакета (`NewAxon`):**

| Поле | Тип | Описание |
|---|---|---|
| `entry_point` | `(u16, u16)` | Координаты на грани (2D, грань плоская) |
| `vector` | `(i8, i8, i8)` | Нормализованный вектор роста |
| `type_mask` | `u8` | Полный тип (Geo + Sign + Variant) для фильтрации дендритами |
| `remaining_length` | `u16` | Сколько ещё может расти аксон |

- **Действие:** Принимающий шард создаёт Ghost Axon — виртуальную структуру, которая растёт от стены внутрь шарда, соблюдая физику коллизий.

**2. Активность (Fast Path / Spikes)**

- **Частота:** Каждый `sync_batch` (часто).
- **Структура пакета (`SpikeBatch`):**

| Поле | Тип | Описание |
|---|---|---|
| `ghost_axon_id` | `u32` | Локальный ID призрачного аксона на принимающем шарде |
| `timestamp` | `u8` | Смещение тика внутри батча |

- **Оптимизация:** Глобальные ID (UUID) не используются. При создании Ghost Axon стороны договариваются о локальном ID (handshake), чтобы в горячем цикле слать только лёгкие `u32`.

### 3.3. Призрачные Аксоны (Ghost Axons)

**Инвариант:** Физика внутри шарда не должна знать, что аксон начался где-то ещё.

- **Сущность:** Ghost Axon — полноценный аксон в памяти принимающего шарда, но без тела (Soma). Его «тело» — порт ввода на границе.
- **Жизненный цикл:**
  1. **Рождается** при получении пакета `NewAxon`.
  2. **Умирает**, если источник присылает сигнал `Prune` или соединение разрывается.
  3. **Цепочка:** Если Ghost Axon дорастает до противоположной стены шарда, он экспортируется в следующий Channel, создавая цепочку Ghost → Ghost → Ghost.

## 4. Алгоритм Роста: Cone Tracing & Steering

Аксон не растёт слепым лучом (Raycast). Это автономный агент, использующий эвристику **Sensing → Weighting → Steering** для максимизации потенциальных связей. Тяжёлая математика, но выполняется **один раз** на этапе Baking (CPU). В GPU-память попадает готовый массив координат.

### 4.1. Фаза 1: Сенсорика (Sensing)

На каждом шаге роста кончик аксона (Head) сканирует пространство перед собой.

- **Инструмент:** Конический запрос (Cone Query) через Spatial Hash на базе PackedPosition (X|Y|Z).
- **Параметры:**
  - `FOV` (Field of View): Угол конуса (например, 60°). Хранится как `fov_cos = cos(FOV/2)` — без тригонометрии в цикле.
  - `Lookahead` (`MAX_SEARCH_RADIUS`): Дистанция «зрения» (например, 50 мкм).
- **Фильтрация:** В выборку попадают только сомы совместимых типов (проверка маски `Type_ID` из `blueprints.toml`). Игнорируются сомы, уже имеющие связь с этим аксоном.
- **Сложность:** O(K) — ищем только в соседних чанках Spatial Grid, не перебираем все N нейронов.

### 4.2. Фаза 2: Взвешивание (Weighting)

Вычисляем вектор локального притяжения (`V_attract`). CPU-only, f32 легален.

```rust
// baking_compiler/src/cone_tracing.rs

pub fn calculate_v_attract(
    head_pos: Vec3,       // f32, текущая координата конца аксона
    forward_dir: Vec3,    // f32, нормализованный вектор роста
    fov_cos: f32,         // cos(FOV / 2), вычислен при старте
    spatial_grid: &Grid,  // Ключи из X|Y|Z PackedPosition
) -> Vec3 {
    let mut v_attract = Vec3::ZERO;
    let mut total_weight = 0.0;

    // 1. Поиск кандидатов в соседних чанках: O(K), не O(N)
    // Защита от OOB: Grid обязан делать clamp(0, MAX) при поиске чанков
    let candidates = spatial_grid.get_in_radius_safe(head_pos, MAX_SEARCH_RADIUS);

    for target in candidates {
        let dir = target.pos - head_pos;
        let dist_sq = dir.length_squared();
        if dist_sq > MAX_RADIUS_SQ { continue; }  // Без sqrt на этапе отсева

        let dist = dist_sq.sqrt();
        let dir_norm = dir / dist;

        // 2. Проверка конуса: Dot Product (без atan2/sin/cos)
        if forward_dir.dot(dir_norm) >= fov_cos {
            // Обратный квадрат расстояния = закон притяжения
            let weight = target.attraction_gradient / (dist_sq + 1e-5);
            v_attract += dir_norm * weight;
            total_weight += weight;
        }
    }

    // 3. Итоговый вектор коррекции
    if total_weight > 0.0 {
        (v_attract / total_weight).normalize()
    } else {
        forward_dir  // Нет целей → расти прямо
    }
}
```

### 4.3. Фаза 3: Руление и Квантование (Steering & Packing)

Итоговый вектор роста — смешение глобальной цели и локального интереса:

```
V_next = Normalize(V_global * W_inertia + V_attract * W_sensor + V_noise * W_jitter)
```

| Компонент | Описание | Роль |
|---|---|---|
| `V_global` (Goal) | Вектор к целевому слою (по Z / топологии проекции) | Гарантирует, что аксон дойдёт до цели |
| `V_attract` (Local) | Тяга к скоплениям нейронов (§4.2) | Создаёт извилистость (tortuosity) |
| `V_noise` (Jitter) | Шум (`WyHash64(master_seed + axon_id + step)`) | Выход из локальных минимумов |

- **Веса (`W_*`):** Задаются в `anatomy.toml`. Инерция 60–80%.

**Дискретный Шаг и Упаковка (CPU Night Phase):**

Математика ведётся в f32, но в память запекается квантованный `u32` (PackedPosition). Позиция f32 сохраняется между шагами для защиты от drift округления.

```rust
// baking_compiler/src/axon_growth.rs

pub fn step_and_pack(
    current_f32_pos: &mut Vec3,   // f32 позиция — предотвращает drift
    v_global: Vec3,
    v_attract: Vec3,
    v_noise: Vec3,
    weights: &SteeringWeights,
    owner_type_mask: u8,          // 4 бита (Geo|Sign|Variant)
    segment_length_voxels: f32,   // 2.0 (50 мкм / 25 мкм)
) -> u32 {
    // 1. Steering (f32)
    let v_steer = (v_global * weights.global
                 + v_attract * weights.attract
                 + v_noise * weights.noise).normalize_or_zero();

    // 2. Step
    *current_f32_pos += v_steer * segment_length_voxels;

    // 3. Quantize → voxel coords (с защитой границ)
    let x = (current_f32_pos.x.round() as u32).min(1023); // 10 бит
    let y = (current_f32_pos.y.round() as u32).min(1023); // 10 бит
    let z = (current_f32_pos.z.round() as u32).min(255);  // 8 бит
    let t = (owner_type_mask & 0x0F) as u32;               // 4 бита

    let packed = (t << 28) | (z << 20) | (y << 10) | x;

    // 4. Защита от Stagnation (Залипание на границе)
    // Если шаг не изменил квантованную координату (уперлись в край сетки),
    // нужно прервать рост, чтобы не плодить фантомные сегменты в одном вокселе.
    assert!(packed != previous_packed_pos, "Stagnation detected: axon stalled at boundary");
    
    packed
}
```

### 4.4. Результат: Smart Wiring

В отличие от рандома, такой аксон:

1. **Огибает пустоты:** Если впереди нет нейронов, идёт прямо по `V_global` (экономит длину).
2. **Ныряет в кластеры:** Если видит группу нейронов, отклоняется, чтобы пройти сквозь неё, максимизируя Overlap Probability для дендритов.
3. **Формирует пучки:** Аксоны, идущие в одну зону, естественным образом сближаются, формируя тракты (как в белом веществе) — их притягивают одни и те же промежуточные цели.
