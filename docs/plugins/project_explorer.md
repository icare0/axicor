# Project Explorer

| Свойство | Значение |
| :--- | :--- |
| **Домен** | `PluginDomain::ProjectExplorer` |
| **Роль** | Файловый навигатор и генератор намерений (Intents) для других плагинов |

## 1. ECS Контракт (Данные)
Плагин опирается на глобальный ресурс `ProjectFsCache` для холодного кэширования дерева файлов воркспейса. 
> [!IMPORTANT]
> Категорически запрещено вызывать `std::fs::read_dir` в горячем цикле отрисовки. Все данные о структуре проекта поставляются только через кэш.

## 2. Шина Событий (Event API)
Project Explorer работает как командный центр, генерируя события для активации других доменов:

| Событие | Триггер | Полезная нагрузка |
| :--- | :--- | :--- |
| `ZoneSelectedEvent` | Клик по `.pos` / `.state` | `{ target_window: Entity, zone_hash: u32 }` |
| `LoadGraphEvent` | Клик по `brain.toml` | `{ target_window: Entity }` |

> [!NOTE]
> Эмиттер (Project Explorer) самостоятельно определяет `target_window` для ивента. Это обеспечивает жесткую маршрутизацию и предотвращает пробуждение всех открытых окон приложения при одном клике.

## 3. Особенности Исполнения (Execution Paths)
* **Cold Path:** Первичная индексация файлов воркспейса и заполнение `ProjectFsCache`.
* **UI Path:** Отрисовка дерева папок в `egui` с использованием отложенного вычисления видимых элементов.
