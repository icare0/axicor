# Node Editor (Shard Assembler)

**Домен:** `PluginDomain::NodeEditor`
**Цель:** Визуальная сборка макро-маршрутизатора коннектома (`brain.toml` и `io.toml`). Никакого визуального программирования логики, только настройка топологии.

## 1. Zero-Lock Data Provider
Парсинг конфигов вынесен в фоновую систему `sync_topology_graph_system`. Она реагирует на междоменное событие `LoadGraphEvent`, читает `brain.toml` и рекурсивно парсит `io.toml` для каждой зоны, складывая результат в ECS-ресурс `BrainTopologyGraph`. UI-поток никогда не блокируется дисковыми операциями.

## 2. Двухпроходный DOD-Рендер (Z-Index Isolation)
Канвас строится на базе `egui::Painter` без использования тяжелых сторонних библиотек. Отрисовка строго разделена на фазы:
1. **Zero-Render Calc:** Вычисление габаритов нод (в зависимости от кол-ва пинов в `io_configs`) и точных 2D-координат портов ввода/вывода на экране.
2. **Background Pass (Кривые Безье):** Отрисовка Ghost Axon Projections (связей) от пина к пину. Линии ложатся на задний фон.
3. **Foreground Pass (Ноды):** Отрисовка тел зон и их пинов. 

## 3. UI State vs Biological State
Экранные координаты нод, зум и панорамирование канваса живут в изолированном ресурсе `NodeGraphUiState`. Они никак не влияют на биологический смысл `BrainConfig`.
