# Node Editor (Shard Assembler)

| Свойство | Значение |
| :--- | :--- |
| **Домен** | `PluginDomain::NodeEditor` |
| **Роль** | Макро-роутинг и визуальная сборка топологии (`brain.toml`). Никакой биологической логики, только чёрные ящики (Зоны) и связи (Ghost Axons). |

## 1. ECS Контракт (Данные)
* **`NodeGraphUiState` (Компонент)**: Локальное состояние окна. Хранит экранные координаты нод (`HashMap<String, Pos2>`), текущий масштаб (`zoom`) и смещение камеры (`pan`). Данные строго изолированы от биологического описания графа.
* **`BrainTopologyGraph` (Ресурс)**: Глобальный кэш разобранной топологии проекта. Хранит списки зон и связей `(From, To)` для быстрой отрисовки без постоянного обращения к диску.

## 2. Шина Событий (Синхронизационный Мост)
Node Editor работает в связке с `Code Editor` и `Project Explorer`, выступая приемником событий обновления графа:

* **Listens to `LoadGraphEvent`**: Эксплицитная загрузка модели при клике на скомпилированный `.axic` архив.
* **Listens to `TopologyChangedEvent`**: Реакция на сохранение кода (кнопка `Apply` в `Code Editor`). Вызывает горячую перерисовку графа.
* **Listens to `OpenFileEvent`**: Пассивное прослушивание (Sniffing). Если пользователь открывает `brain.toml` в `Project Explorer`, нодовый редактор перехватывает путь и автоматически загружает соответствующий граф.

> **[DOD Invariant]**: При получении любого из этих событий, плагин асинхронно перепарсивает `brain.toml` и обновляет ресурс `BrainTopologyGraph`, не блокируя UI-поток.

## 3. Особенности Исполнения (Execution Paths)

### Горячий путь (Infinite Canvas Rendering)
Для обеспечения идеального Z-индекса и поддержки Pan/Zoom, отрисовка выполняется через низкоуровневый `egui::Painter` в три строгих прохода (3-Pass Render):

1. **Calc Pass:** Перевод локальных координат нод в абсолютные экранные `egui::Rect` с учетом применения матриц масштабирования (`zoom`) и панорамирования (`pan`).
2. **Background Pass (Ghost Axons):** Отрисовка межзональных связей в виде кубических кривых Безье. Отрисовываются первыми, чтобы визуально уйти на задний план под ноды.
3. **Foreground Pass (Nodes):** Отрисовка плашек зон, заголовков и регистрация невидимых зон взаимодействия (`egui::Sense::drag()`) для перемещения нод пользователем.
