# Node Editor (Shard Assembler)

| Свойство | Значение |
| :--- | :--- |
| **Домен** | `PluginDomain::NodeEditor` |
| **Роль** | Макро-роутинг и визуальная сборка топологии (`brain.toml`). Никакой биологической логики, только чёрные ящики (Зоны) и связи (Ghost Axons). |

## 1. ECS Контракт (Данные)
* **NodeGraphUiState (Компонент)**: Локальное состояние окна. Хранит экранные координаты нод, масштаб (zoom), смещение (pan), буферы переименования и флаги видимости модальных окон (`show_clear_modal`).
* **BrainTopologyGraph (Ресурс)**: Глобальный менеджер сессий. Содержит `sessions: HashMap<PathBuf, ProjectSession>`.
* **ProjectSession (Структура)**: Локальный кэш конкретного открытого графа. Хранит списки зон, закэшированные ID (`zone_ids`), связи и флаг `is_dirty` для отложенной компиляции.

## 2. Шина Событий (Синхронизационный Мост)
Node Editor выступает приемником и генератором:
* **Listens to LoadGraphEvent / OpenFileEvent**: Асинхронная подгрузка файлов графа в фоновом пуле (`AsyncComputeTaskPool`) с парсингом TOML.
* **Listens to EntityDeletedEvent**: Вычищает открытые `ProjectSession` и сбрасывает фокус, если директория графа была физически удалена.
* **Emits TopologyMutation**: Отправка намерений в шину. Мы перешли на строгую DTO-маршрутизацию. Используются только три глобальных интента: `Create(CreateTarget)`, `Delete(DeleteTarget)` и `Rename(RenameTarget)`. Оркестраторы WM перехватывают их для каскадного изменения файлов на диске.
* **Emits OpenContextMenuEvent**: Делегирование отрисовки ПКМ-меню в унифицированную систему оконного менеджера.

## 3. Особенности Исполнения (Execution Paths)

### Горячий путь (Infinite Canvas Rendering)
Для обеспечения идеального Z-индекса и поддержки Pan/Zoom, отрисовка выполняется через низкоуровневый `egui::Painter` в три строгих прохода (3-Pass Render):

1. **Calc Pass:** Перевод локальных координат нод в абсолютные экранные `egui::Rect` с учетом применения матриц масштабирования (`zoom`) и панорамирования (`pan`).
2. **Background Pass (Ghost Axons):** Отрисовка межзональных связей в виде кубических кривых Безье. Отрисовываются первыми, чтобы визуально уйти на задний план под ноды.
3. **Foreground Pass (Nodes):** Отрисовка плашек зон, заголовков и регистрация невидимых зон взаимодействия (`egui::Sense::drag()`) для перемещения нод пользователем.
