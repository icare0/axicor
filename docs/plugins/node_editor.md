# Node Editor (Shard Assembler)

| Свойство | Значение |
| :--- | :--- |
| **Домен** | `PluginDomain::NodeEditor` |
| **Роль** | Сборка макро-топологии (brain.toml, io.toml). Только роутинг, никакой биологической логики. |

## 1. ECS Контракт (Данные)
* **NodeGraphUiState (Компонент)**: Хранит экранные координаты нод, текущий зум и панорамирование канваса. Данные строго изолированы от биологического описания графа.
* **BrainTopologyGraph (Ресурс)**: Глобальный кэш разобранной топологии проекта.

## 2. Шина Событий (Event API)
* **Listens to:** `LoadGraphEvent`.
* **Action:** При получении события активирует фоновую систему `sync_topology_graph_system` для парсинга TOML-файлов. Парсинг выполняется асинхронно, не блокируя UI-поток.

## 3. Особенности Исполнения (Execution Paths)

### Горячий путь (Rendering)
Отрисовка выполняется через `egui::Painter` в три этапа (Multi-pass DOD):
1. **Zero-Render Calc:** Расчет габаритов нод на основе количества портов в `io.toml` и вычисление экранных координат пинов.
2. **Background Pass:** Отрисовка кривых Безье (*Ghost Axons*), соединяющих порты.
3. **Foreground Pass:** Отрисовка тел нод и интерактивных элементов интерфейса.
