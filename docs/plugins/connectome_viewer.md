# Connectome Viewer (3D)

**Домен:** `PluginDomain::Viewport3D`
**Цель:** Высокопроизводительная инспекция 3D-геометрии шардов (сомы, аксоны) без просадок FPS.

## 1. Zero-Copy Загрузка (Data Pipeline)
Геометрия не парсится построчно. Бинарный файл `shard.pos` читается в RAM, и сырые байты мгновенно кастуются в `&[u32]` через `bytemuck::cast_slice`. 
* **Dynamic AABB Centering:** Лоадер делает предварительный проход для вычисления Bounding Box шарда. Геометрия аппаратно сдвигается так, чтобы центр масс облака точек всегда находился строго в `Vec3::ZERO`. Это исключает параллакс-иллюзии при вращении камеры.
* **Garbage Collection:** Старые меши выжигаются из ECS перед загрузкой новых через маркерный компонент `ShardGeometry { viewport: Entity }`. Никаких утечек VRAM.

## 2. Изоляция Оптики и Инпутов
Камера плагина **ничего не знает** о глобальной мыши или системных событиях ОС. 
Она питается строго компонентом `PluginInput`, который оконный менеджер собирает при взаимодействии с RTT-текстурой (Render-To-Texture) конкретного тайла.
* **Scroll Quantization:** Дельта колеса мыши (`raw_scroll_delta.y`) сбрасывается до `.signum()` и применяется как логарифмический шаг (15%), защищая от смертельных скачков камеры.
* **WGPU Safe-Start:** Камера рождается мертвой (`is_active = false`), пока WM не аллоцирует для нее VRAM бэкбуфер. Это исключает `Camera order ambiguities` и рендер поверх системного окна.

## 3. Roadmap: Hardware Instancing
Отрисовка 100 000+ сом через `PrimitiveTopology::PointList` (1-пиксельные точки) — это legacy. Запланирован переход на аппаратный инстансинг: в памяти лежит 1 сфера, а позиции/цвета отдаются в GPU через плоский буфер. Строго 1 Draw Call.
